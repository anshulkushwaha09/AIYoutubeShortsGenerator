## Step 7 â€” GitHub Actions (Daily Automation)

The workflow at `.github/workflows/daily_shorts.yml` runs every day at **10:00 AM UTC**.

### Add GitHub Secrets

Go to your repo â†’ **Settings â†’ Secrets and variables â†’ Actions â†’ New repository secret**

| Secret Name         | Value                                     |
| ------------------- | ----------------------------------------- |
| `GEMINI_API_KEY`    | Your Gemini API key                       |
| `PEXELS_API_KEY`    | Your Pexels API key                       |
| `YOUTUBE_TOKEN_B64` | Base64-encoded `token.pickle` (see below) |

### Encode token.pickle

```powershell
# Run this in PowerShell in the project root after completing Step 4:
[Convert]::ToBase64String([IO.File]::ReadAllBytes('token.pickle')) | clip
```

This copies the encoded token to your clipboard. Paste it as the `YOUTUBE_TOKEN_B64` secret.

### Enable GitHub Actions

1. Push your code to GitHub (make sure `.env` and `token.pickle` are in `.gitignore` âœ…)
2. Go to **Actions tab** â†’ the workflow appears automatically
3. Click **Run workflow** to test it manually first

## Troubleshooting

| Error                          | Fix                                                                |
| ------------------------------ | ------------------------------------------------------------------ |
| `GEMINI_API_KEY not set`       | Check `.env` file exists and key is correct                        |
| `PEXELS_API_KEY not set`       | Check `.env` file exists and key is correct                        |
| `token.pickle not found`       | Run `python auth_youtube.py` first                                 |
| `client_secret.json not found` | Download from Google Cloud Console (Step 1)                        |
| FFmpeg not found               | Ensure FFmpeg is installed and on your PATH                        |
| Upload fails on GitHub Actions | Re-run `auth_youtube.py` and update the `YOUTUBE_TOKEN_B64` secret |
