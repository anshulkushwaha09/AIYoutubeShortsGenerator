# Security Policy

## Supported Versions

We currently support the latest version of the project. Since this is an active open-source tool, we recommend always pulling the latest changes from the `main` branch.

| Version | Supported          | Notes           |
| ------- | ------------------ | --------------- |
| 1.0.x   | :white_check_mark: | Current Release |
| < 1.0   | :x:                | Deprecated      |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in this project (e.g., API key leaks, unsafe file handling, or dependency issues), please report it responsibly.

### How to Report

**Do not open a public issue.** Instead, please email the maintainer directly or reach out via a private message:

- **Email:** [Your Email Here - Optional]
- **GitHub Security Advisory:** [Use the "Report a vulnerability" button on the repository Security tab]

### What to Include

Please include:

1.  Description of the vulnerability.
2.  Steps to reproduce the issue.
3.  Any relevant screenshots or logs.

### Response Timeline

- We will acknowledge your report within **48 hours**.
- We will send a fix or a timeline for the fix within **1 week**.

Thank you for helping keep this project safe!
